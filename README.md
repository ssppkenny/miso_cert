# cert

A Java OCP (Oracle Certified Professional) exam practice quiz application built in Haskell using the [Miso](https://haskell-miso.org/) framework.

## Overview

`cert` lets you practice Java certification exam questions organized by chapter. It covers 14 chapters of the OCP study guide plus three full practice exams. The application runs entirely in the browser and follows the Elm Architecture (Model–Update–View) as provided by Miso.

## Features

- **Chapter navigation** — 14 chapters covering Java Fundamentals, Operators, Core APIs, Exceptions, Collections, Streams, Modules, Concurrency, I/O, JDBC, Localization, and three Practice Exams.
- **Question types** — Single-answer (radio buttons) and multiple-answer (checkboxes, detected automatically from "Choose two/three/..." in question text).
- **Code blocks** — Java code samples are rendered in a styled monospace block.
- **Images** — Per-question and per-option images served from `resources/chapterN/` and rendered inline.
- **Answer persistence** — Selected answers are stored in browser `localStorage` keyed by chapter, so progress survives page refreshes.
- **Score evaluation** — Results page with score (correct/total, percentage), color-coded banner (green ≥80%, yellow ≥60%, red <60%), per-question ✓/✗ grid, wrong vs. correct answer details, and explanation tooltips.

## Building and Running

### Native build (recommended for development)

The native build uses `jsaddle-warp`: Haskell runs on a local server and communicates with the browser via WebSocket.

```bash
stack build
stack exec cert-exe
# Open http://localhost:3000
```

Set the `PORT` environment variable to change the port (default: `3000`).

For an optimized build:

```bash
stack build --ghc-options='-O2'
```

For a static binary suitable for deployment:

```bash
stack build --ghc-options='-optl-static -optl-pthread'
```

See [DEPLOYMENT.md](DEPLOYMENT.md) for deployment details.

### WASM build

Requires the GHC WASM toolchain. Install it once:

```bash
curl https://gitlab.haskell.org/haskell/ghc-wasm-meta/-/raw/master/bootstrap.sh \
  | GHCUP_SKIP_UPDATE_CHECK=1 sh
source ~/.ghc-wasm/env
```

Then build:

```bash
./build-wasm.sh            # outputs to ./dist/
./build-wasm.sh --serve    # build and serve on http://localhost:8080
```

Or use the Makefile inside the Nix WASM shell:

```bash
nix develop .#wasm
make        # build + optimize
make serve  # serve public/ on http://localhost:8080
```

> **Note:** The WASM build compiles successfully but the app does not render to the DOM at runtime due to a `JSM`→`IO` bridge issue between Miso and the GHC WASM backend. The native build is recommended. See [WASM-STATUS.md](WASM-STATUS.md) for details.

### Nix dev shells

```bash
nix develop          # default GHC shell
nix develop .#hls    # with Haskell Language Server
nix develop .#wasm   # WASM toolchain
nix develop .#ghcjs  # GHCJS toolchain
```

## Data Format

**`resources/chapters.txt`** — one entry per line:
```
1:Java Fundamentals
2:Operators
...
```

**`resources/chapterN/questions.txt`** — numbered questions with lettered options:
```
1.  Question text here
    (optional Java code block)
A.  First option
B.  Second option
...
```

**`resources/chapterN/answers.txt`** — correct answer keys with explanations:
```
1.  A.  Explanation text here.
```

**`resources/chapterN/images.txt`** — list of image filenames present in the chapter directory (one per line, auto-generated by `make build`).

## Dependencies

| Package | Purpose |

| `mtl` | State monad utilities |
